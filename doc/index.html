Basic info

Dcm is an interpretted, concatenative, embedded, object oriented, scripting language. It borrows ideas from many different languages such as Python, Factor, and Javascript. The name comes from the original intent of the language which was supposed to be a simple extention of the ubiquitous unix utility dc (which now ships with bc). DCM was to mean dc improved, however the language has since diverged greatly, and with reason as can be seen below with a sample factorial program in each language.

dc:[d1-d1>f*]sf 6 lfx
DCM:[ dup 1 - dup 0 > fac [] if ex * ] $fac 6 fac ex

Why should I use DCM

The best answer to this is, "It don't know"e;. The language was desiged to some very specific needs and wants I had for a project. When I was begun work on this project, I could not find a project that met these requirements. Some of the key high level requirements are listed below; if you need a mojority of them, then perhaps DCM is a good choice for you.
  • It shall be concatenative
  • It shall run quickly with a low memory footprint
  • It shall be design to be embedded into other C++ applications
  • It shall have exception handling
  • It shall have functional features
